Unreal Engine Gameplay框架

2026年6月25日 | 来源：Epic Games 官方文档

概述

虚幻引擎的Gameplay框架是一组类，你可以基于它们提供的模块化功能构建自己的游戏体验。你可以从中选取适合自己游戏的元素，因为这些类是相互配合、相辅相成的。

Gameplay框架为游戏开发提供了核心架构，包括游戏模式、玩家状态、控制器、Pawn、摄像机等关键组件。这些类共同构成了游戏逻辑的基础，让开发者能够高效地构建复杂的游戏系统。

核心框架类

1. 游戏实例 (Game Instance)

UGameInstance 会在引擎启动时实例化，并保持活跃直至引擎关闭。这是一种管理器类，在游戏中不存在实体，但会追踪数据和运行代码。

  • 生命周期：从引擎启动到关闭，跨越所有关卡
  • 网络特性：在联网多人游戏中不会被复制，独立存在于服务器和所有客户端上
  • 主要用途：管理关卡加载间需要保持的数据，如游戏保存系统
  • 子系统管理：可以创建和管理游戏实例子系统，如在线子系统（好友、游戏会话、排行榜等）

2. 游戏模式 (Game Mode)

AGameModeBase 会在引擎加载关卡并创建世界后立即实例化。游戏模式是一种基于服务器的管理器类，继承自Actor类。

  • 生命周期：关卡加载时创建，按地图一一设置
  • 核心地位：在Gameplay框架中处于核心地位
  • 主要职责：管理Gameplay会话的所有规则和结构
  • 创建顺序：是关卡加载时第一个被实例化的Actor，创建时实例化剩余的框架Actor

3. 游戏状态 (Game State)

AGameStateBase 是非实体Actor，用于追踪游戏的整体状态。

  • 数据内容：包含与游戏内所有玩家相关的数据和逻辑
  • 具体示例：队伍得分、游戏目标、所有玩家及其相关玩家状态的列表
  • 网络特性：在联网多人游戏中，会在权威服务器和所有互联的客户端间复制其状态信息
  • 创建方式：由游戏模式创建

4. 玩家状态 (Player State)

APlayerState 处理其关联玩家的相关数据和逻辑。

  • 数据内容：玩家特定的数据，如生命值、子弹数量、物品栏
  • 创建时机：在每个玩家加入游戏或进入关卡时分别创建
  • 网络特性：在联网多人游戏中，会在权威服务器和所有互联的客户端间复制其状态信息

5. 控制器 (Controller)

控制器类处理玩家在游戏世界内的操作逻辑。UE中有两种用途广泛的控制器类：

  • 玩家控制器 (APlayerController)：管理器类，处理来自人类的输入，显示抬头信息并处理游戏内的物理表现
  • AI控制器 (AAIController)：管理器类，处理游戏内的物理表现，在UE的人工智能帮助下显示其操作，包括行为树、状态树、寻路等

控制器类及其派生的类在游戏世界内没有实体，作为非实体Actor类存在。

6. Pawn

APawn 类由玩家在游戏世界内的物理实体组成，对玩家的重要性不亚于控制器类。

  • 物理实体：构成玩家在游戏世界中的物理表现形式
  • 控制关系：控制器处理Pawn并指使其在游戏执行操作
  • 组件构成：由数种Actor组件构成，包括碰撞组件、静态网格体组件和移动组件
  • 基类关系：派生自Actor类

7. 角色 (Character)

ACharacter 是派生自Pawn的子类，专门用于实现具有移动能力的角色。

  • 默认Pawn：由默认Pawn类以及多种功能丰富的组件构成
  • 核心组件：角色移动组件、骨架网格体组件和胶囊体组件
  • 适用场景：玩家角色或NPC等需要复杂移动和动画的对象

3C概念

虚幻引擎中的3C概念构成了角色控制和视觉表达的核心要素：

  • Controller（控制器）：用于接收来自输入设备的输入信号，Controller并不直接控制角色的物理运动，而是通过与对应的Pawn进行交互，发送命令和更新状态。主要用于控制Pawn，通过输入信号又可以控制玩家角色，然后配合相机，从游戏获取信息，控制Character的反馈。
  • Camera（摄像机）：实时为玩家显示游戏中画面。设置Camera与Controller配合，可以实现第一人称视角（FPV）、第三人称视角（TPV）和其他多种视角模式，同时也可以调整相机的视野、远近裁剪面、缩放等因素，以增强玩家的游戏体验。
  • Character（角色）：游戏中代表的玩家或者非玩家实体对象(AI控制)。Character类通常是从Pawn类派生而来，Character通常带有骨骼动画系统支持，能够进行复杂的动作表现，并且与物理系统相结合，进行运动和碰撞检测。

综上所述，"3C"构成了虚幻引擎中角色控制和视觉表达的核心要素，它们相互协同工作，共同实现了游戏中角色的交互行为及玩家的视觉感知。

代码示例

游戏模式示例

// 自定义游戏模式头文件 #pragma once #include "CoreMinimal.h" #include "GameFramework/GameModeBase.h" #include "MyGameMode.generated.h" UCLASS() class MYGAME_API AMyGameMode : public AGameModeBase { GENERATED_BODY() public: AMyGameMode(); // 游戏开始时调用 virtual void BeginPlay() override; // 玩家登录时调用 virtual void PostLogin(APlayerController* NewPlayer) override; // 游戏结束逻辑 virtual void HandleMatchHasEnded() override; };

角色初始化示例

// 自定义角色构造函数 AMyCharacter::AMyCharacter() { // 设置胶囊体大小 GetCapsuleComponent()->InitCapsuleSize(42.f, 96.0f); // 让相机不做旋转 bUseControllerRotationPitch = false; bUseControllerRotationYaw = false; bUseControllerRotationRoll = false; // 角色在输入方向时进行旋转 GetCharacterMovement()->bOrientRotationToMovement = true; // 控制角色转向时的速率 GetCharacterMovement()->RotationRate = FRotator(0.0f, 500.0f, 0.0f); // 跳跃高度 GetCharacterMovement()->JumpZVelocity = 700.f; // 跳跃向前浮动移动速率 GetCharacterMovement()->AirControl = 0.35f; // 最大移动速度 GetCharacterMovement()->MaxWalkSpeed = 500.f; // 创建相机臂组件 CameraBoom = CreateDefaultSubobject(TEXT("CameraBoom")); CameraBoom->SetupAttachment(RootComponent); // 创建跟随相机 FollowCamera = CreateDefaultSubobject(TEXT("FollowCamera")); FollowCamera->SetupAttachment(CameraBoom, USpringArmComponent::SocketName); }
