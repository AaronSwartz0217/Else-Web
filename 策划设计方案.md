# 策划设计方案

## 1. 项目定位与目标

### 1.1 项目定位
本项目是一个基于 Three.js 的 3D 交互式个人展示网站，主要用于：
- 介绍作者个人信息和专业背景
- 展示作者的作品集和项目案例
- 提供沉浸式的用户体验
- 通过隐藏游戏增加网站的趣味性和互动性

### 1.2 目标用户
- 潜在雇主和合作伙伴
- 行业同行和技术爱好者
- 对作者作品感兴趣的访问者

### 1.3 核心价值
- 独特的 3D 视觉体验
- 创新的滚轮式交互方式
- 专业的作品展示平台
- 趣味性的隐藏游戏环节

## 2. 整体架构设计

### 2.1 页面结构

| 页面名称 | 功能描述 | 技术实现 |
|---------|---------|----------|
| 主页面 | 作者介绍、作品集展示 | Three.js 3D 场景 |
| 游戏页面 | 互动游戏体验 | Three.js 3D 场景 + 交互控制 |

### 2.2 技术架构

```
前端技术栈：
├── HTML5 + CSS3 + JavaScript

```

## 3. 页面布局与视觉设计

### 3.1 主页面设计

#### 3.1.1 场景布局
- **初始场景**：星空背景，中央显示作者名称和欢迎文字
- **作者介绍区**：滚动进入后显示作者详细介绍和专业技能
- **作品集展示区**：以 3D 卡片形式展示作者的项目作品
- **联系信息区**：显示作者的联系方式和社交账号
- **游戏入口**：滚动到场景尽头时触发游戏页面加载

#### 3.1.2 视觉风格
- **色调**：深色背景配合高对比度的元素，营造科技感和专业感
- **字体**：现代无衬线字体，简洁易读
- **动画**：平滑的相机移动，卡片旋转效果，文字淡入淡出
- **光影**：使用 Three.js 的光照系统，增强 3D 效果的真实感

### 3.2 游戏页面设计

#### 3.2.1 场景布局
- **游戏场景**：3D 环境，包含可交互的模型和元素
- **控制说明**：底部显示操作指南
- **返回按钮**：允许用户返回主页面

#### 3.2.2 视觉风格
- **延续主页面的设计风格**：保持一致的视觉语言
- **游戏元素**：根据游戏类型设计相应的 3D 元素和特效
- **反馈效果**：为用户操作提供即时的视觉反馈

## 4. 交互流程设计

### 4.1 主页面交互流程

1. **初始状态**：用户进入网站，看到欢迎界面和开始按钮
2. **滚轮交互**：
   - 向上滚动：相机向前移动，进入作者介绍区
   - 继续向上滚动：进入作品集展示区，卡片逐渐清晰
   - 继续向上滚动：进入联系信息区
   - 滚动到尽头：触发游戏页面加载
3. **卡片交互**：
   - 鼠标悬停：卡片放大并高亮
   - 点击卡片：显示作品详情或链接到作品页面
4. **导航控制**：
   - 提供返回顶部的按钮
   - 可能的快捷导航菜单

### 4.2 游戏页面交互流程

1. **进入游戏**：从主页面滚动到尽头后自动加载
2. **游戏控制**：
   - WASD 键：移动相机
   - 鼠标：旋转视角
   - 滚轮：缩放视角
3. **游戏目标**：
   - 探索 3D 环境
   - 与模型互动
   - 可能的小游戏挑战
4. **返回主页面**：
   - 提供明显的返回按钮
   - 可能的游戏结束后自动返回

```javascript
// 创建作品卡片
function createPortfolioCards() {
    const cardGeometry = new THREE.PlaneGeometry(2, 3);
    
    portfolioData.forEach((item, index) => {
        // 加载作品图片
        const texture = textureLoader.load(item.image, () => {
            console.log(`Texture loaded for ${item.title}`);
        }, undefined, (error) => {
            console.error(`Error loading texture for ${item.title}:`, error);
        });
        
        // 创建卡片材质
        const cardMaterial = new THREE.MeshStandardMaterial({
            map: texture,
            transparent: true,
            opacity: 1,
            side: THREE.DoubleSide,
            roughness: 0.3,
            metalness: 0.2
        });
        
        const card = new THREE.Mesh(cardGeometry, cardMaterial);
        // 卡片定位
        const side = index % 2 === 0 ? -3 : 3;
        card.position.x = side;
        card.position.z = -Math.floor(index / 2) * 6;
        card.position.y = Math.sin(index * 0.1) * 0.5;
        
        // 添加卡片到场景
        cards.push(card);
        scene.add(card);
    });
}
```

```javascript
// 初始化游戏场景
function initGameScene() {
    scene = new THREE.Scene();
    scene.background = new THREE.Color(0x000000);
    
    camera = new THREE.PerspectiveCamera(75, window.innerWidth / window.innerHeight, 0.1, 1000);
    camera.position.set(0, 2, 5);
    
    renderer = new THREE.WebGLRenderer({ antialias: true });
    renderer.setSize(window.innerWidth, window.innerHeight);
    document.getElementById('canvas-container').appendChild(renderer.domElement);
    
    // 添加轨道控制器
    controls = new THREE.OrbitControls(camera, renderer.domElement);
    controls.enableDamping = true;
    controls.dampingFactor = 0.05;
    
    // 添加光照
    addGameLights();
    
    // 加载游戏模型
    loadGameModels();
}
```

```javascript
// 键盘事件处理
function onKeyDown(event) {
    switch (event.code) {
        case 'KeyW':
            moveForward = true;
            break;
        case 'KeyS':
            moveBackward = true;
            break;
        case 'KeyA':
            moveLeft = true;
            break;
        case 'KeyD':
            moveRight = true;
            break;
        case 'Escape':
            // 返回主页面
            window.location.href = 'index.html';
            break;
    }
}

// 相机移动更新
function updateCamera() {
    if (moveForward) {
        camera.translateZ(-speed);
    }
    if (moveBackward) {
        camera.translateZ(speed);
    }
    if (moveLeft) {
        camera.translateX(-speed);
    }
    if (moveRight) {
        camera.translateX(speed);
    }
}
```

## 7. 内容规划

### 7.1 作者介绍内容
- 个人简介：姓名、职业、专业背景
- 技能展示：专业技能、技术栈、工具熟练度
- 教育背景：学历、相关课程
- 工作经历：实习或项目经验

### 7.2 作品集内容
- 项目案例：每个项目的名称、描述、技术栈、成果
- 作品分类：可以按照类型（Web 开发、3D 设计、游戏开发等）分类
- 作品详情：每个作品的详细介绍、截图或演示

### 7.3 联系信息内容
- 联系方式：邮箱、电话
- 社交账号：GitHub、LinkedIn、个人博客等
- 简历下载：提供 PDF 简历下载链接

### 7.4 游戏内容
- 游戏类型：基于 3D 模型的探索游戏
- 游戏目标：探索环境、与模型互动
- 游戏机制：简单的解谜或收集元素

## 8. 优化与扩展建议

### 8.1 性能优化
- **模型优化**：使用低多边形模型，减少三角形数量
- **纹理优化**：压缩纹理图片，使用适当的分辨率
- **渲染优化**：使用 frustum culling，只渲染相机可见的物体
- **加载优化**：实现资源的预加载和懒加载

### 8.2 用户体验优化
- **响应式设计**：确保在不同设备上的良好显示
- **加载状态**：添加加载动画，提升用户等待体验
- **错误处理**：优雅处理资源加载失败的情况
- **辅助功能**：添加键盘导航和屏幕阅读器支持

### 8.3 功能扩展
- **作品筛选**：添加按类型、技术栈等筛选作品的功能
- **作品详情页**：为每个作品创建详细的介绍页面
- **博客集成**：添加技术博客或文章分享功能
- **多语言支持**：添加中英文切换功能
- **深色/浅色模式**：支持不同的主题模式

### 8.4 游戏扩展
- **游戏内容**：增加更多的游戏元素和互动性
- **游戏关卡**：设计多个游戏关卡，增加游戏深度
- **得分系统**：添加游戏得分和排行榜功能
- **保存功能**：允许用户保存游戏进度

## 10. 结论

本策划设计方案为基于 Three.js 的 3D 交互式个人展示网站提供了详细的规划和实施指导。通过采用新颖的滚轮式交互方式和沉浸式的 3D 视觉效果，网站将为访问者提供独特的浏览体验，有效展示作者的个人信息和作品集。

同时，通过在场景尽头加载隐藏游戏的设计，增加了网站的趣味性和互动性，使整个浏览过程更加生动有趣。

该方案考虑了技术实现的可行性、用户体验的优化以及未来的扩展可能性，为项目的成功实施奠定了基础。
